사용에 따라이 지침을 공식화 되는 F# 에서 큰 코드 베이스 프로그래머의 다양 한 그룹을 사용 합니다. 이 지침은 일반적으로 잠재 고객을 성공적으로 사용 F# 하 고 시간이 지남에 따라 프로그램에 대 한 요구 사항 변경 시 불만 최소화 합니다.

## <a name="five-principles-of-good-f-code"></a>좋음이 다섯 가지 원칙 F# 코드

염두에 다음과 같은 원칙이 작성할 때마다 F# 시간이 지남에 따라 변경 되는 시스템에서 특히 코드입니다. 모든 부분은 향후 문서에서 지침이 5 지점에서 기인합니다.

1. **좋은 F# 코드는 간결 하 고 인상적 이며 구성 가능**

    F#에 더 적은 줄의 코드로 작업 express 및 일반 기능을 다시 사용할 수 있도록 하는 많은 기능이 있습니다. F# 핵심 라이브러리도 많은 유용한 형식 및 일반적인 데이터 컬렉션을 사용 하 여 작업에 대 한 함수를 포함 합니다. 사용자 고유의 함수 및에 결합 하는 F# 핵심 라이브러리 (또는 다른 라이브러리)는 자연 스러운 루틴의 일환 F# 프로그래밍 합니다. 일반적으로 더 적은 줄 코드의 문제에 솔루션을 표현할 수 있으면 다른 개발자 (또는 이후 자신) 됩니다 감사의. 또한 좋습니다 FSharp.Core의 같은 라이브러리를 사용 하는 합니다 [방대한.NET 라이브러리](../../../api/index.md) 는 F# 실행 또는 타사 패키지에 [NuGet](https://www.nuget.org/) 중요 한 작업을 수행 해야 하는 경우.

2. **좋은 F# 코드는 상호 운용이 가능**

    상호 운용성 등 서로 다른 언어로 코드를 사용 하는 여러 개의 폼을 걸릴 수 있습니다. 다른 호출자에 게 상호 운용 하는 코드의 경계가 중요 한 것도 이기도 한 경우 호출자의 권한을 가져오려면 F#입니다. 작성 하는 경우 F#에 항상 생각할 있습니다와 같은 다른 언어에서 그렇게 하는 경우를 포함 하 여, 작성 하는 코드를 호출 하는 방법을 다른 코드에 대 한 C#합니다. 합니다 [ F# 구성 요소 디자인 지침](component-design-guidelines.md) 자세히 상호 운용성에 설명 합니다.

3. **좋은 F# 코드에서는 개체 프로그래밍을 하지 개체 방향**

    F#.NET 개체를 사용한 프로그래밍에 대 한 전체 지원에 포함 [클래스](../language-reference/classes.md), [인터페이스](../language-reference/interfaces.md)합니다 [액세스 한정자](../language-reference/access-control.md), [추상 클래스](../language-reference/abstract-classes.md)등에입니다. 컨텍스트 인식 해야 하는 함수와 같은 더 복잡 한 함수 코드에 대 한 개체 컨텍스트 정보 함수 없는 방식으로 쉽게 캡슐화 됩니다. 같은 기능 [선택적 매개 변수](../language-reference/members/methods.md#optional-arguments) 하 고 신중 하 게 활용할 [오버 로드](../language-reference/members/methods.md#overloaded-methods) 쉽게 수행할 수 있습니다이 기능의 소비 호출자에 대 한 합니다.

4. **좋은 F# 코드 mutation 노출 하지 않고 잘 수행**

    아닙니다. 고성능 코드를 작성 하려면 변형을 사용 해야 합니다. 컴퓨터의 작동 원리, 결국 이며 이러한 코드는 이해 하기 어렵고 오류가 발생 하기 쉬운 경우가 많습니다. Mutation 호출자에 게 노출 하지 마십시오. 대신 [변이 기반 구현을 숨기는 기능 인터페이스를 구축](conventions.md#performance) 성능이 중요 한 경우.

5. **좋은 F# 코드는 사항에 신경 쓰지**

    작업 대규모 코드 베이스를 작성할 수 있습니다에 대 한 도구는 유용 F# 사용 하 여 보다 효율적으로 사용할 수 있도록 코드 F# 언어 도구입니다. 예로 있는지 확인 하면 지나치면 지점 무료 스타일이 프로그래밍, 디버거를 사용 하 여 중간 값을 검사할 수 있도록 합니다. 또 다른 예로 사용 하 여 [XML 문서 주석](../language-reference/xml-documentation.md) 편집기의 도구 설명 호출 사이트에서 이러한 의견을 표시할 수 있도록 구문을 설명 합니다. 항상 어떻게 코드, 탐색, 디버깅를 읽고 다른 프로그래머가 자신의 도구를 사용 하 여 조작 하는 방법에 대 한 생각 합니다.
